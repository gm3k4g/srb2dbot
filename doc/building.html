sudo pacman -S cmake dpp nlohmann-json

Fedora

sudo dnf install cmake gcc-c++ dpp-devel json-devel

Building

Using the Build Script (Recommended)

The build.sh script auto-detects NixOS vs regular Linux and handles everything:

# Default: Debug build with tests
./build.sh

# Run tests after build
RUN_TESTS=ON ./build.sh

# Release mode
BUILD_TYPE=Release ./build.sh

# Custom build directory
BUILD_DIR=out ./build.sh

Manual Build

# Configure
cmake -B build -DCMAKE_BUILD_TYPE=Release

# Build (without tests)
cmake --build build -j$(nproc)

# Build with developer mode (tests, linting)
cmake -B build -DCMAKE_BUILD_TYPE=Debug -Dsrb2dbot_DEVELOPER_MODE=ON
cmake --build build -j$(nproc)

Testing

The project includes 52 unit tests covering utility functions, filename sanitization, script manipulation, chat bridge message sanitization, file polling, emoji conversion, and integration-level bridge cycle simulation.

# Build with tests
BUILD_DIR=build BUILD_TYPE=Debug RUN_TESTS=ON ./build.sh

# Or manually
cmake -B build -DCMAKE_BUILD_TYPE=Debug -Dsrb2dbot_DEVELOPER_MODE=ON
cmake --build build
cd build && ctest --output-on-failure

# Run individual test binary with verbose output
./build/test/srb2dbot_test

Manual Bridge Integration Test

After deploying the bot with a configured secret.json:

  1. Discord → SRB2: Send a message in the configured bridge Discord channel, then check ~/.srb2/luafiles/client/DiscordBot/discordmessage.txt for the sanitized output.
  2. SRB2 → Discord: Write a test line to ~/.srb2/luafiles/client/DiscordBot/Messages.txt (e.g., echo "Player1: test" >> Messages.txt). Within 2 seconds the message should appear in the bridge Discord channel with mentions suppressed.

CMake Presets

The project provides several CMake presets for CI environments:

  • ci-ubuntu — GCC/clang on Ubuntu with clang-tidy and cppcheck
  • ci-macos — AppleClang on macOS
  • ci-win64 — MSVC on Windows
  • ci-sanitize — Debug build with ASan and UBSan
  • ci-coverage — Coverage build with gcov/lcov
cmake --preset ci-ubuntu
cmake --build build -j$(nproc)

Configuration

After building, create a secret.json file:

cp secret.json.default secret.json
FieldRequiredDescription
bot_token Yes Discord bot token from Developer Portal → Bot → Token
bot_id Yes Bot's own Discord user ID (Application ID). Prevents chat bridge echo
guild_id Yes Discord server (guild) ID for slash command registration
channel_id For bridge Channel ID for chat relay. Set to "0" or omit to disable bridge
service_name For server control systemd user service name (default: srb2@srb2b)

Important: Enable Message Content Intent in the Discord Developer Portal under Bot → Privileged Gateway Intents for the chat bridge to function.
