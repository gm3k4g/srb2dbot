srb2dbot

A Discord bot that remotely controls Sonic Robo Blast 2 game servers. Built with the D++ library in C++26, it manages server lifecycle, script editing, WAD file uploads, and in-game administration — all through Discord slash commands.

Quick Start

# Clone and build
git clone https://github.com/gm3k4g/srb2dbot.git
cd srb2dbot

# On NixOS / with Nix installed
./build.sh

# On regular Linux
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j$(nproc)

# Configure
cp secret.json.default secret.json
# Edit secret.json with your bot token and guild ID

How It Works

The bot communicates with an SRB2 server through a named FIFO pipe (~/.srb2/srb2_servers.d/srb2b.d/srb2b.fifo). SRB2 reads commands from this pipe as if they were typed in the server console.

Server lifecycle (start, stop, restart) is managed through systemd user services, with the service name configurable via secret.json.

Script editing operates on the bash configuration file (~/.srb2/srb2_servers.d/srb2b.d/srb2b.sh), validated with bash -n before writing.

Features at a Glance

CategoryCommandsDescription
Script 7 View, search, insert, remove, change, and move lines in the server config script
WADs 4 List, search, and upload addon files via Discord attachment or URL
Server 3 Restart, stop the server, and send raw console commands
Players 3 Send chat messages, kick, and ban players

Security Model

  • All slash commands require p_ban_members permission
  • Pipe input is sanitized — newlines, carriage returns, and control characters are rejected
  • Script validation uses fork/exec with argument arrays instead of shell string construction
  • Uploaded files are capped at 100 MB with filename path-traversal protection
  • Password lines in scripts are redacted automatically

Dependencies

  • C++26 compiler (GCC 14+ or Clang 18+)
  • CMake 3.14+
  • D++ — Discord API library
  • nlohmann/json — JSON parsing
  • systemd — for service management
  • bash — for script validation

Test Suite

The project includes 28 unit tests covering utility functions, filename sanitization, and all script manipulation logic. Run with: 

BUILD_DIR=build BUILD_TYPE=Debug RUN_TESTS=ON ./build.sh
