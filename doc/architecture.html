Runtime Pipeline

Commands flow through three distinct paths:

1. Script Path

Script editing commands (/get_script, /find_line, /insert_line, etc.) read and write ~/.srb2/srb2_servers.d/srb2b.d/srb2b.sh. Each write is validated with bash -n via fork() + execlp() before committing.

Core logic is extracted into source/script.cpp as pure string-in/string-out functions, making them independently testable without filesystem access.

2. WAD Path

WAD commands (/list_wads, /search_wads, /addfile_upload, /addfile_link) operate on ~/.srb2/addons/. Uploads are capped at 100 MB and filenames are sanitized to prevent path traversal attacks.

3. Server Pipe Path

Server and player commands (/server_do, /server_say, /kick_player, /ban_player, /restart_server, /stop_server) communicate with SRB2 through:

  • FIFO pipe at ~/.srb2/srb2_servers.d/srb2b.d/srb2b.fifo for console commands. All input is sanitized to reject control characters and newlines.
  • systemd via systemctl restart|stop <service> --user for service lifecycle. The service name is configurable in secret.json.

Key Files

FileRole
source/main.cppBot entry point. Initializes D++, reads secrets, registers slash commands, runs the event loop. Contains anonymous namespace functions for filesystem operations that depend on POSIX APIs (FIFO pipes, directory iteration, process management).
source/utils.cppPure utility functions: sanitize_filename() strips path components from user input, link_filename_str() extracts filenames from URLs.
source/script.cppScript manipulation logic operating on string content rather than files. Includes script_inspect_str(), script_find_str(), script_move_line_str(), script_add_line_str(), script_remove_line_str(), and script_change_line_str().
secret.json.defaultConfiguration template — copy to secret.json and fill in bot token, guild ID, and service name.
source/version.h.cmakeCMake template that generates version.h with project metadata and command name strings.

Security Architecture

Attack VectorMitigation
Command injection via script pathfork() + execlp() with argument arrays — shell never interprets the path
Command injection via pipe inputControl characters (\x00-\x1F except tab), newlines, and carriage returns are rejected before writing to the FIFO
Path traversal via upload filenamessanitize_filename() extracts only the filename component via std::filesystem::path::filename(), rejects . and ..
Unbounded upload sizeDownloads are capped at 100 MB; response bodies exceeding MAX_WAD_SIZE are rejected
JSON parse exceptionstry/catch around json::parse() with early file-open check
Information disclosure (passwords)Lines containing "password" are redacted in script output
FIFO truncationwrite() retry loop handles partial writes and EAGAIN

CI Pipeline

CMake presets provide automated builds for Ubuntu (GCC/clang with clang-tidy, cppcheck), macOS (AppleClang), and Windows (MSVC). Dev mode enables CTest, clang-format linting, codespell, and code coverage with gcov/lcov.
